# Certificate Canonical Rules

## Canonical Filename

**Frozen:** `certificate.v2.4.0.json`

## Canonical Ordering

### Key Ordering
- Alphabetical keys (case-sensitive)
- UTF-8 encoding
- No whitespace significance

### Serialization Rules
- JSON format only
- No alternative serializations
- No XML, YAML, or other formats
- No compressed formats (gzip, etc.)

## Prohibited

- Alternative filenames
- Alternative serializations
- Format variations
- Compression

## Validation

Certificates must:
- Use filename: `certificate.v2.4.0.json`
- Use alphabetical key ordering
- Use UTF-8 encoding
- Use JSON format only

## Authority

These rules are frozen for v2.4.0. No changes allowed.
