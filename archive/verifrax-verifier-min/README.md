# Historical Minimal Verifier Archive

This directory preserves the earlier `verifrax-verifier-min` surface as historical repository material.

It is not an active maintained verifier.
It is not the canonical minimal execution target.
It must not be treated as a current reference implementation unless and until a maintained minimal verifier is reintroduced under the active `verifier/` hierarchy.

## Active verifier authority

The active maintained verifier surfaces are:

- `verifier/node`
- `verifier/rust`

These are the only current maintained verifier directories in the repository authority structure.

## Status

- Classification: archival
- Maintenance state: inactive
- Normative authority: none
- Operational role: historical preservation, lineage continuity, audit support

## Interpretation rule

Consumers, implementers, and verification tooling must not depend on this archived directory for active protocol conformance, release verification, or maintained execution semantics.

## Scope note

This archive is retained to preserve earlier minimal-verifier experiments, partial implementations, or superseded repository structure without allowing that historical material to remain ambiguous with current verifier authority.
