# Issue & PR Policy

## Authoritative Repositories

**GitHub Issues DISABLED on:**
- `Verifrax/VERIFRAX` (frozen)
- `Verifrax/VERIFRAX-SPEC` (frozen)
- `Verifrax/VERIFRAX-verify` (frozen)

## Non-Authoritative Repositories

**GitHub Issues ENABLED on:**
- `Verifrax/VERIFRAX-DOCS` (documentation only)
- `Verifrax/VERIFRAX-SAMPLES` (examples only)

## Issue Restrictions

**BANNED:**
- Feature requests
- Roadmap discussions
- "Why" debates
- Design discussions (use design repo)

## Pull Request Policy

**Authoritative repos:**
- PRs require maintainer approval
- No direct pushes
- Protected branches

**Non-authoritative repos:**
- PRs allowed for documentation/examples
- No authority claims

## Design Repository

- Issues allowed for design discussions only
- No implementation code
- No authority
