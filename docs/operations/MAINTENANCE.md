# Maintenance Mode

**Status: MAINTENANCE ONLY**

## Accepted changes

- Security patches
- Checksum updates (with version bump)
- Documentation corrections
- New freeze snapshots (append-only)
- Signed releases

## Rejected changes

- New features
- Engine modifications (without new freeze)
- Schema changes (without major version)
- Certificate format changes

## Change process

1. Open issue describing change
2. Reference checksum files affected
3. Create PR with version bump
4. Update BUILD_HASH.txt
5. Create new freeze snapshot
6. Create signed release
