# Language Lock

**Naming Matters. Words Create Reality.**

## Banned Terms (Internal)

These words are **banned** from VERIFRAX codebase:

- ❌ "verify" - Implies checking, not finalizing
- ❌ "check" - Implies validation, not adjudication
- ❌ "validate" - Implies approval, not finality

## Required Terms

These words **must** be used:

- ✅ "finalize" - Creates finality
- ✅ "contradict" - Detects narrative collision
- ✅ "invalidate" - Applies retroactive invalidation
- ✅ "adjudicate" - Produces verdict

## Rationale

**"Verify"** suggests a process that can be repeated or overridden.  
**"Finalize"** creates irreversible state.

**"Check"** implies temporary validation.  
**"Adjudicate"** produces permanent judgment.

**"Validate"** suggests approval workflow.  
**"Invalidate"** applies structural power.

## Enforcement

- Code reviews must reject banned terms
- Documentation must use required terms
- APIs must use required terms
- Error messages must use required terms
