# VERIFRAX Primitive Topics Standard

This document freezes the canonical GitHub topics standard for the Verifrax primitive layer.

This topics standard is normative for:

- GitHub repository topics
- public discovery surfaces
- ecosystem indexing surfaces
- future cataloging surfaces

## Canonical topics set

Every primitive repository in this program must use exactly these topics:

- verifrax
- verifrax-protocol
- protocol-primitive
- deterministic-systems
- irreversible-systems
- cryptographic-governance
- artifact-integrity
- supply-chain-security

## Rules

1. Every primitive repository must use the full canonical topics set.
2. Topic spelling must match exactly.
3. Topic order should remain stable wherever ordering is preserved.
4. No repository-specific vanity topic may replace a canonical topic.
5. Additional topics may be added later only through an approved governance update, not ad hoc repo drift.
6. The canonical topics set exists to unify public discovery and ecosystem identity across all eight primitives.

## Constraint

No primitive repository public topic surface may contradict this topics standard.
