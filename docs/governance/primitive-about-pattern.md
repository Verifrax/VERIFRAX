# VERIFRAX Primitive About Pattern

This document freezes the canonical GitHub About pattern for the Verifrax primitive layer.

This About pattern is normative for:

- GitHub repository About descriptions
- npm package short descriptions
- top-level public registry surfaces
- ecosystem listings
- future discovery surfaces

## Canonical About sentence pattern

Every primitive repository must use this sentence pattern:

Verifrax primitive — <role phrase> for deterministic irreversible systems.

## Rules

1. Every primitive repository must use the same sentence structure.
2. Only the role phrase may vary between primitives.
3. The prefix must remain exactly `Verifrax primitive —`.
4. The suffix must remain exactly `for deterministic irreversible systems.`
5. No marketing language, hype language, or alternate slogan may replace the canonical pattern on the primary About surface.
6. npm short descriptions should follow the same pattern unless a strict character limit requires a minimal shortening that preserves the same role identity.

## Constraint

No primitive repository, package description, or public discovery surface may contradict this About pattern.
