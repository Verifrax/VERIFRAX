# VERIFRAX Release History

This directory preserves historical release materials for VERIFRAX.

It exists to retain lineage continuity, audit traceability, and earlier release-era repository surfaces without allowing historical materials to remain ambiguous with active repository authority.

This directory is historical only.

## Purpose

`release-history/` preserves:

- historical freeze-era manifests
- earlier release snapshots
- archival release-bound artifacts
- superseded release structures retained for audit continuity

These materials support reconstruction, lineage inspection, and long-horizon repository verification history.

## Active authority boundary

The active release-integrity authority surface is:

- `release-integrity/`

The active frozen release-surface definition is:

- `release-integrity/freeze-surfaces.json`

Historical material under `release-history/` must not be interpreted as the active source of release authority.

## Versioned historical layout

Historical materials are preserved under versioned subdirectories such as:

- `release-history/v1/`

Each versioned directory may contain:

- preserved freeze surfaces
- archival manifests
- historical release notes
- superseded release-bound repository artifacts

## Interpretation rule

`release-history/` is retained for lineage continuity and audit support only.

It does not define active protocol semantics, active maintained verifier authority, active conformance targets, or active release-integrity authority.
