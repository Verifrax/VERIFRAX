What VERIFRAX Is

VERIFRAX is a deterministic software verification service that:

  • Accepts payment via Stripe Checkout Session (one-time payment, EUR)
  • Mints execution tokens upon successful payment confirmation (cryptographically signed, single-use)
  • Accepts evidence bundles (binary files) with a verification profile identifier
  • Executes deterministic verification using a specified verifier version
  • Generates certificates (canonical JSON documents) containing evidence bundle hash, verification profile, verifier version, execution timestamp, verdict, and certificate hash
  • Stores certificates in immutable storage at content-addressed paths
  • Serves certificates via public HTTP endpoint using certificate hash

What VERIFRAX Is Not

  • Not a truth verification service — VERIFRAX does not verify the truth, accuracy, or legal validity of evidence
  • Not an advisory service — VERIFRAX provides no interpretation, guidance, or recommendations
  • Not a custody service — VERIFRAX does not hold, manage, or control user funds beyond payment processing
  • Not a general-purpose storage service — Evidence bundles are retained solely as a by-product of execution finality and certificate reproducibility
  • Not a dispute resolution service — VERIFRAX does not resolve disputes, mediate, or arbitrate
  • Not a certification authority — VERIFRAX does not issue certificates of authenticity, compliance, or legal standing
  • Not a guarantee service — VERIFRAX makes no guarantees about outcomes, acceptance, or third-party reliance

Determinism & Finality: One payment → one execution → one certificate. Identical inputs produce identical outputs (byte-for-byte). No randomness, no interpretation, no discretion. Execution is irreversible.

Non-Reliance: No party may rely on a VERIFRAX certificate as a substitute for independent verification, legal judgment, or regulatory determination.

Execution Flow

  1. Create payment intent (select tier A or B)
  2. Complete payment via Stripe Checkout
  3. Receive execution token (single-use, time-limited)
  4. Upload evidence bundle with profile identifier
  5. Receive certificate hash
  6. Retrieve certificate via /certificate/{hash}

→ Execute Verification
